Academic Integrity

AI Use Rules For This Assessment

Permitted support

  • Clarify UI terms such as Golden Ratio, Rule of Thirds, HTML, and layout.
  • Brainstorm layout ideas or improvement directions.
  • Improve grammar, structure, and formatting.
  • Learn basic syntax or tool steps.
  • Generate placeholder content that you adapt and justify.

Not permitted

  • Submitting AI-generated final answers without your own understanding.
  • Using AI to replace your app analysis, screenshots, design decisions, or client feedback.
  • Submitting a full AI-generated design as if it was your own design work.
  • Creating fake observations, fake screenshots, or fake feedback.
Key principle: AI can support your process, but all analysis, design, development, evidence, and explanations must reflect your own work.

Task 1

Applied Knowledge Questions: Interface Design Context

Task 1 checks whether you understand important UI concepts and can explain how they are used in practical interface design work.

Assessment Text

Task 1 - Applied Knowledge Questions (Interface Design Context)

You are working as a junior UI designer in a small digital product team. Your role is to support the design and development of user interfaces that are visually effective, functional, and aligned with web standards.

For each of the following scenarios, provide a clear and practical response based on your understanding of interface design and development concepts.

Each answer must:

  • Be written in your own words
  • Demonstrate applied understanding (not only theory)
  • Include examples where relevant
Explanation

Use simple language. Define the concept, show how it applies to a UI, then give a practical example. Avoid copying generic definitions only.

Evidence Checklist - Task 1 Overall

T1 Q1

Layout and Visual Balance: Golden Ratio

Explain what the Golden Ratio is and how it can help a landing page feel balanced.

Assessment Text

Q1. Layout and Visual Balance (Golden Ratio)

You are designing a landing page for a new digital product. Your team wants the layout to feel visually balanced, professional, and easy to navigate.

Explain:

  • What the Golden Ratio is (in your own words)
  • How it can be applied to structure a user interface layout
  • Provide one practical example of how you would use it in your landing page (e.g. layout sections, spacing, image/text balance)

Answer requirements: At least 100 words.

Explanation

The Golden Ratio is a proportion often described as about 1:1.618. In UI, you can use it as a guide for width, spacing, image/text balance, or section hierarchy. You do not need to do complex maths, but you should show how proportion can make a layout feel balanced.

Text area 38% Hero image 62% CTA Next section

Safe answer structure

  1. Define the Golden Ratio in your own words.
  2. Explain how designers use proportion to organise visual weight.
  3. Give one landing page example such as a 38/62 split, larger hero image, or spacing rhythm.
  4. Explain how this helps users scan and navigate the page.

T1 Q2

Image Composition in UI: Rule of Thirds

Select or create an image and show how the Rule of Thirds supports image placement and text visibility.

Assessment Text

Q2. Image Composition in UI (Rule of Thirds)

You are selecting a hero image for the homepage of a website. The goal is to make the image visually engaging while ensuring key content (e.g. text, buttons) is clearly visible.

You are required to:

  • Select or create an image (any image suitable for a UI)
  • Apply the rule of thirds by dividing the image into 9 equal sections
  • Position key elements of the image according to this rule

Then explain:

  • How you applied the rule of thirds
  • Why this improves the effectiveness of the image in a user interface

Answer requirements: Image with visible grid or clear explanation. At least 100 words.

Explanation

The Rule of Thirds divides an image into three columns and three rows. Important subjects often sit near the lines or intersections. In UI, this helps you place text, buttons, and focal points without making the image feel crowded.

Left third Centre third Right third CTA safe space

What to submit

  • A selected or created image.
  • A visible 3 by 3 grid over the image, or a very clear written explanation.
  • A short explanation of where the main subject and UI text/button are placed.
  • A reason this improves visibility, focus, and visual interest.

T1 Q3

HTML in User Interface Development

Choose at least three HTML tags and explain how they structure a UI.

Assessment Text

Q3. HTML in User Interface Development

You are working with a web developer to build a basic interface for a website. You are asked to suggest key HTML elements that will structure the content of the page.

Provide:

  • At least three (3) HTML elements (tags) commonly used in UI development
  • For each:
    • Write the HTML code
    • Explain what it does
    • Explain how it contributes to the structure or functionality of a user interface

Answer requirements: At least 100 words.

Explanation

Pick tags you understand. A strong answer does not list tags only. It explains how each tag helps a page become readable, structured, accessible, or interactive.

HTML tag optionWhat it doesHow it helps UI
<header>Defines the top section of a page or area.Helps users find branding, navigation, or the main heading quickly.
<nav>Groups navigation links.Makes movement between pages or sections clearer.
<button>Creates an interactive button.Supports actions such as submit, continue, open, or save.
<form>Groups input fields for user information.Creates the structure for sign-up, contact, or search workflows.
<section>Defines a meaningful page section.Improves content organisation and scanning.

Assessment Text

Task 2 - Establish Requirements and Build a UI Prototype

You are working as a Junior UI Designer in a digital product team.

Your team has been asked to improve an existing mobile application that currently has usability and interface issues.

Your trainer will act as your client, providing direction and feedback throughout the project.

Your goal is to:

  • analyse an existing application
  • identify UI issues
  • design an improved interface
  • build a prototype
  • refine it based on client feedback

Project Instructions (Must be followed)

  1. Select a mobile application you have used at least twice
  2. The application must have a clear purpose and show obvious UI issues or areas for improvement
  3. Present your choice to your trainer (client) and obtain approval
  4. Select a prototyping tool (e.g. Figma, Canva, Adobe XD or similar)
  5. Confirm your tool choice with your trainer
Explanation

This is a practical project. Keep screenshots, notes, prompt iterations, client feedback, and final prototype evidence as you work. Do not leave evidence collection until the end.

1. Select appUsed at least twice and has clear UI issues.
2. Analyse UIScreenshots, issues, users, purpose.
3. PrototypePrompt, generate, test, iterate.
4. RefineGet client feedback and improve.

Evidence Checklist - Task 2 Overall

T2 Q1

Application Analysis

Show what the current app does, who uses it, and which interface areas need improvement.

Assessment Text

Q1 - Application Analysis (Understanding the Current UI)

Provide the following information about your selected mobile application:

  • Application Name
  • What the application does (main function and purpose)
  • Target users (who typically uses this app)
  • Screenshot(s) of the current interface
  • Clearly indicate (highlight or describe) the areas you plan to improve

Important: Your screenshots must show real UI issues or opportunities for improvement.

Answer requirements: At least 100 words + screenshots at least 2.

Explanation

Do not only say "the app is confusing." Be specific. For example: the button is hard to see, the form labels are unclear, there are too many steps, the icons are not obvious, or the layout does not group related content.

Questions to help your analysis

PurposeWhat task does this app help users complete?
Target usersWho uses this app, and what do they need quickly?
UI issuesWhere do users need to tap, read, search, or decide?
EvidenceWhich two screenshots clearly show the issue?

T2 Q2

UI Project Requirements

Define the problem, your improvement plan, expected outcome, and organisational guidelines.

Assessment Text

Q2 - UI Project Requirements (Defining the Problem and Solution)

You are now required to define your UI improvement project clearly.

Explain:

  • What specific UI issues or problems exist in the current application (e.g. navigation, layout, readability, user flow, interaction issues)
  • How you will improve the interface (be specific - layout, components, interaction, structure)
  • What you expect your improved UI to achieve (e.g. better usability, faster interaction, clearer content)
  • What organisational guidelines you will consider (e.g. branding, colours, consistency, accessibility, user expectations, ratings)

Important: Your response must show clear reasoning and practical decision-making.

Answer requirements: At least 100 words.

Explanation

This question is about design reasoning. Show the connection between issue, change, and expected user benefit. Organisational guidelines can include brand colours, tone, accessibility, privacy, consistency with app patterns, app store expectations, or rating/review concerns.

ProblemImprovementExpected result
Navigation is hidden or unclear.Use clearer labels, visible tabs, or simplified menu structure.Users find key actions faster.
Text is hard to read.Increase contrast, font size, spacing, and section hierarchy.Users understand content with less effort.
Forms are long or confusing.Group fields, improve labels, add helpful microcopy.Users complete tasks with fewer errors.

T2 Q3

AI-Assisted Prototype Development and Design Refinement

Use Replit, Lovable, or Bolt to generate and refine your improved UI through prompt iterations.

Assessment Text

Q3 - AI-Assisted Prototype Development and Design Refinement

You are now required to design and build your improved UI using an AI-powered prototype development tool.

Instead of manually creating a prototype, you will use AI "vibe coding" tools to generate your interface using prompts.

Step 1 - Select an AI Tool

  • Replit - Builds working apps from prompts, including UI, logic, and hosting
  • Lovable - Generates full web applications from simple descriptions (ideal for beginners)
  • Bolt - Fast browser-based tool for generating front-end and full-stack apps

Step 2 - Develop Your Improved Prototype

  • use prompt engineering to generate your improved UI
  • ensure your design reflects the improvements identified in Q2
  • includes colour and visual hierarchy
  • shows clear layout and structure
  • improves usability and clarity

Step 3 - Document Your Prompting Process

  • all prompts used
  • all iterations (changes to prompts)
  • AI-generated outputs at each stage
  • screenshots OR screen recording (video)

Step 4 - Client Review

  • present your prototype to your trainer (client)
  • receive feedback
  • refine your design based on that feedback

What you must submit: final prototype, prompt engineering evidence, client feedback and refinement, and a written explanation minimum 100 words.

Explanation

Do not only submit the final screen. The assessment wants to see the process: prompt, output, problem noticed, refined prompt, improved output, client feedback, final changes.

Assessment Text

Task 3 - Design and Build User Interface

You are working as a UI Designer responsible for creating a website for a digital product or service.

This may be a project you have developed, an idea you are exploring, or a concept you want to bring to life.

Your goal is to design and build a clear, visually engaging, and functional user interface that:

  • communicates your idea effectively
  • uses relevant media (images, visuals, or video where appropriate)

Project Requirements

  • Two pages only: Landing Page and Sign-up Page

  • A clear call to action leading users to sign up
Explanation

The website should be simple and complete. Two pages is enough: a persuasive landing page and a sign-up page with the required form fields.

T3 Q1

Content Flow: User Journey and Interaction

Describe how a user moves from the landing page to completing the sign-up.

Assessment Text

Q1 - Content Flow (User Journey and Interaction)

Describe the content flow of your website from when a user first lands on the page to when they complete the sign-up.

Your response must:

  • Be written in dot points
  • Include at least 5 steps
  • Clearly describe:
    • what the user sees
    • what the user does
    • how the interface guides them
  • Demonstrate a logical and clear user journey

Answer requirements: At least 100 words.

Explanation

Write this as a journey. Each dot point should include a screen/section, user action, and interface guidance. For example: "The user sees the hero section, reads the value proposition, and clicks the sign-up button."

1. ArriveUser sees product name and value proposition.
2. UnderstandUser reads benefits and how it works.
3. DecideUser sees CTA and trust/contact details.
4. Sign upUser completes form and submits.
