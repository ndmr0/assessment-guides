Useful Research Sources

Use current and reliable sources. AI can help you understand terms, but your demographics and platform analysis should come from real sources.

Official platform sources

Meta, Facebook, Google, YouTube Help, YouTube Blog, and official platform policy or product pages.

Usage reports

Digital reports, social media statistics reports, industry research, and market overview articles.

User behaviour

Articles about video consumption, creator behaviour, short-form video, recommendations, comments, and engagement patterns.

UX testing methods

Nielsen Norman Group, usability testing guides, persona guides, journey mapping guides, and research planning resources.

Testing tools

Google Forms, Microsoft Forms, Zoom, Teams, screen recording tools, Otter.ai, Notion, Figma, FigJam, or Miro.

Evidence notes

Record the title, author or organisation, date, link, and the specific point you used from each source.

Evidence Checklist

Task 2 | Q1

Define Target Market And User Types

Assessment Text
Research the users of Facebook and YouTube using reliable sources. Identify the target market for each platform and define at least two user types for each platform. The target market refers to general user characteristics such as age groups, location, and usage trends. User types describe how users behave, based on experience, behaviour, and interaction with the platform.
Explanation

This question has two parts. First, complete the platform table with researched target markets and two user types for each platform. Second, explain the criteria for each user type. Do not confuse demographics with behaviour. "18-34 year olds" is target market information; "passive video viewer" is a user type.

Good user type language:

Passive viewer, content creator, frequent commenter, short-form video user, low digital literacy user, experienced platform user, new user, mobile-only viewer.

Evidence Checklist

Task 2 | Q2

Create User Personas

Assessment Text
Create a user persona for each user type. Each persona should be a realistic profile based on behaviour and research. Each persona should reflect how the user behaves, their level of experience, and how they interact with the platform. Include name, age, location, gender, and personality.
Explanation

You need four personas: two for Facebook and two for YouTube. Each persona must match one of your user types from Q1. Keep the persona realistic and useful for testing. The persona should help you decide what tasks, questions, and problems matter for that user.

Persona check:

If your user type is "new YouTube user", the persona should not behave like an expert creator. The name, age, experience, goals, and frustrations should all fit the user type.

Evidence Checklist

Task 2 | Q3

Customer Journey Mapping

Assessment Text
Analyse and visually represent the user journey for the video feature on both platforms. Create one customer journey map for Facebook and one customer journey map for YouTube. The journey should show how a user interacts with the platform from discovering content, watching or interacting with video, and continuing engagement. Submit screenshots or exported visuals created in Figma or a similar tool.
Explanation

This is a visual task. You need two journey maps, one for Facebook and one for YouTube. Each map should show stages, user actions, thoughts or feelings, and problems or frustrations. The problems you identify here should help you write your hypotheses in Q4.

DiscoverHow does the user find the video?
WatchWhat happens while watching?
InteractDoes the user like, comment, share, save, subscribe, or follow?
ContinueWhat does the user do after the video?

Evidence Checklist

Task 2 | Q4

Develop UX Hypotheses

Assessment Text
Create one testable hypothesis for each platform. A hypothesis is a clear statement that predicts user behaviour, identifies a possible issue in the experience, and can be tested during usability testing. Each hypothesis must be based on your research, personas, and customer journey maps.
Explanation

You need two hypotheses: one for Facebook and one for YouTube. A good hypothesis is specific, testable, and linked to a journey map problem. Avoid vague statements like "users do not like the app." State who, what problem, and what you expect to observe during testing.

Useful structure:

If [user type] tries to [video task], then they may experience [specific problem] because [reason from research or journey map]. This can be tested by asking participants to [task] and observing [measure or behaviour].

Evidence Checklist

Task 2 | Q5

UX Testing Action Plan

Assessment Text
Plan how your testing sessions will be conducted. Complete the agency action plan for Facebook and YouTube. Define at least three criteria for selecting five users for each platform, design a three-question screener survey, design a three-question testing survey, plan at least two realistic tasks, and identify at least two tools or pieces of equipment.
Explanation

This is your practical testing plan. You are not just listing ideas. You are explaining how you would recruit suitable users, screen them, run the test, ask feedback questions, and collect data. Keep Facebook and YouTube in separate columns if the template asks for it.

RecruitWho qualifies for each platform test?
ScreenWhich three questions confirm they match your criteria?
TestWhich realistic tasks will they complete?
MeasureWhich questions and tools will capture evidence?
